# Decision record for why use ADR

## Status

accepted

## Context

The project could grow over time and other members could join us, as it is meant to be open source.
We must provide a way to understand the core concepts of the project and the current state of architectural decisions to simplify onboarding.

## Decision

Use ADR files to store architectural decisions and their statuses, based on
https://github.com/joelparkerhenderson/architecture-decision-record/tree/main/locales/en/templates/decision-record-template-by-michael-nygard

## Consequences

Easy onboarding process for new contributors.
Decisions recorded in text — fewer disputes.
